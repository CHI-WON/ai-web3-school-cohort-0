### Security quiz 验证

| Q# | 结果 | 关键 |
|----|------|------|
| Q1 Reentrancy | ✓ | Checks-Effects-Interactions：状态清零在外部调用之前 |
| Q2 tx.origin | ⚠️ | tx.origin = 调用链最外层 EOA。msg.sender = 当前跳的直接调用者。钓鱼场景取前者被绕过 |
| Q3 Flash Loan + MEV | 补课 | 闪电贷利用交易原子性（借+还在一笔交易内），Sandwich = 抢跑→用户成交→尾随 |
| Q4 Agent 私钥管理 | 补课 | 五层安全分层：建议→事实→Policy→Simulation→人工确认。大额永远 Layer 5 |

### Q3/Q4 补课要点

**Flash Loan**：借和还在同一笔交易里。如果中间亏了 → 整笔回滚，贷款方零风险。用于套利/清算攻击。

**Agent 私钥五层模型**：
```
L1 建议 → L2 事实 → L3 Policy → L4 Simulation → L5 人工确认
```
X 发帖：L2。小额 swap：L3-L5。大额转出：永远是 L5。

### DeFi

五个积木：

| 积木 | 核心 | 跟 Security 的连接 |
|------|------|--------------------|
| Token | ERC-20 + decimals + 权限检查 | 假 token 攻击面 |
| AMM | x×y=k + 滑点 + 无常损失 | Sandwich 攻击 |
| Lending | 抵押→借款→清算（预言机触发） | Flash Loan 清算攻击 |
| Stablecoin | 法币抵押/超额抵押/算法 | 脱锚 → 系统性风险 |
| Liquidity | 有价格≠能成交 | 流动性浅 → 预言机可操纵 |

**DeFi quiz 4 题**：清算攻击机制、Token 陷阱（decimals）、稳定币脱锚 vs 系统性风险、Agent DeFi 边界判断（发提醒✅ 追加抵押品❌ 卖币还贷❌）。

### Oracle（预言机）

链外数据 → 链上的桥。三层：

| 层 | 例子 | 风险 |
|----|------|------|
| Price Feed | ETH/USD 喂价 | 延迟→错误清算 |
| Data Feed | 储备证明、利率、体育结果 | 数据源不可验证 |
| AI Oracle | 模型判断（"地址是否高风险"） | 不可追溯、不可争议 |

**AI Oracle 关键**：输入可追溯、模型版本+prompt 记录、输出可复核、允许挑战/仲裁、错误有惩罚。

### Indexing（索引）

链上数据 → 结构化查询。三层方案：Event Indexing → Subgraph → Data Pipeline。

> "RPC 不是数据库。" 查单条用 RPC，查用户所有历史用 Indexing 层。

AI Agent 的链上上下文来自 Indexing 层——Agent 拿到的必须是**带来源、带时间戳、可回溯**的数据。
