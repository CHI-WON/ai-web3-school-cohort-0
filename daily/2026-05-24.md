## 密码学（Cryptography）

**第一性原理**：链上身份不由平台发放，由你能否证明自己控制某个私钥决定。

```
私钥 → 公钥 → 地址
  │      │      └─ 接收资产
  │      └─ 验证签名
  └─ 签名授权（绝不公开）
```

| 概念 | 做什么 | 能公开吗 |
|------|--------|---------|
| 私钥 | 签名 = 授权资产操作 | ❌ 泄漏 = 资产归零 |
| 公钥 | 验证签名来源 | ✅ |
| 哈希 | 数据指纹，验证完整性 | ✅ |
| 签名 | 「我同意」的数学证明 | ✅ |

**连 Security**：tx.origin（签名发起者）≠ msg.sender（当前调用者），这是 tx.origin 攻击的物理基础。

## 钱包（Wallet）

**第一性原理**：钱包管理的是链上控制权，不是「账号资料」。

三层动作，三个风险级别：
```
连接钱包 → 读地址           → 零风险
签名消息 → 证明身份          → 中风险
发送交易 → 改变链上状态     → 高风险
```

**交易生命周期**：点击按钮 → 钱包弹窗 → 签名（最后防线）→ mempool（MEV 发生地）→ 区块打包 → EVM 执行（Reentrancy 发生地）→ 成功/失败

**连 Security**：
- owner 是单个 EOA = 单点爆破
- Agent 不能直接持有私钥——签名权是最后一道人工防线
- Simulation 只模拟此刻状态，mempool 状态变化挡不住

---

## 网络（Network）

**第一性原理**：区块链按区块节奏推进，交易要经过 mempool（公开）→ 共识 → 执行。

```
钱包签名 → RPC → mempool（公开！MEV 搜索者狙击）→ 区块打包 → 共识 → EVM 执行
```

| 概念 | 跟安全的关系 |
|------|------------|
| Block | 交易顺序影响结果 → MEV 根源 |
| Mempool | 交易公开可见 → 三明治攻击发生地 |
| Gas | 攻击者愿付更高 gas 抢跑 |
| L2 | 降低费用，但引入排序器/桥接信任假设 |

**MEV 三明治攻击**：看到你的交易 → 抢跑买入推高价格 → 你的交易在更差价格成交 → 尾随卖出赚差价。

## Security

五个知识节点 + 三条盲区补充：

| 节点 | 核心 | 防护 |
|------|------|------|
| Reentrancy | 外部调用未完成时被重入 | Checks-Effects-Interactions |
| Access Control | owner 是 EOA？有 timelock？ | 多签 + timelock + event |
| Audit | 不是安全保证书 | 查 commit、权限、经济机制 |
| Simulation | 签名前预演 | 挡明显错误，挡不住 mempool 变化 |
| Monitoring | 上线后感知异常 | 监控 + 响应闭环 |

盲区补充：
- **tx.origin**：永远用 msg.sender
- **Flash Loan**：无抵押借款 → 一笔交易内操纵价格
- **Agent 私钥**：Agent 不应直接持有私钥，五层安全分层

## 收获

### 全局心智模型

Web3 安全需要从底层理解——不能单独看 Security 章节。每一层都是攻击面：

```
密码学层：私钥泄漏、签名伪造
钱包层：EOA 单点爆破、用户看不懂签名内容
网络层：mempool 抢跑、区块重组、L2 排序器风险
合约层：Reentrancy、Access Control、升级风险
安全层：Audit + Simulation + Monitoring 闭环
```

### 关键直觉

1. **mempool 是公开的** — MEV 不是 bug，是透明性的代价
2. **msg.sender ≠ tx.origin** — 合约间调用链让两者分离
3. **审计 ≠ 安全** — 只覆盖特定 commit，没说明升级权限 = 不完整
4. **Simulation + Audit + Monitoring** — 三者缺一不可
