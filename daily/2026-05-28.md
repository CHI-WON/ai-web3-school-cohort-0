## Bridge 章节

### Machine Payment（机器支付）
Agent/API/服务/钱包之间的自动报价→授权→付款→收据→预算控制。
- 预算多层拆解：全局 → 任务 → 单次 → 服务方 → 紧急停止
- Quote：服务内容 + 价格 + 币种 + 收款地址 + 有效期 + 交付条件 + 退款 + quote id
- Payment Intent：绑定任务/金额/收款方/有效期/quote——是审计的锚点
- x402/MPP：机器间支付协议化
- 危险设计：只设总预算不设频率限制 → Agent 可把全部额度花在一次异常报价上

### Settlement & Escrow（结算与托管）
钱什么时候释放、服务怎么算完成、失败怎么退款、争议怎么处理。
- Escrow 状态机：Created → Funded → Delivered → Accepted/Refunded/Disputed
- Delivery Proof：文件 hash / API 日志 / 模型输出签名 / TEE attestation
- Acceptance：AI 初审 + challenge window + 人工复核（高价值任务不能只靠模型）
- Dispute：争议成本必须 < 争议金额，否则小额任务无法维权
- Evaluator：脚本查格式 + 测试查功能 + AI 查语义 + 人类处理争议——不全压给一个模型

### Quiz 结果
Machine Payment: 3/3 都对（gas buffer / quote 有效期 vs 链上确认时间 / 付款流程缺 quote+receipt）
Settlement & Escrow: 2/3（Q3 小额 dispute 经济学不熟，补充：保证金 > 争议金额 = 无维权 incentive）

## Proposal 初稿

文件：`hackathon/proposal-draft.md`

**方向**：Agent Service Marketplace
**赛道**：Agentic Commerce / Payment
**核心场景**：Alice Agent 花 1 USDC 委托 Bob Agent 翻译一篇中文周报

完整支付闭环（Bridge 六层骨架）：
```
Chain-aware Context  → Agent 读取余额、授权、预算
Web3 Tool Use        → 只读查报价 / 写入 escrow（权限分离）
Agent Workflow       → 状态机：quote→funded→delivered→accepted
Agent Wallet         → Session Key：单次 ≤5 USDC / 每天 ≤20 USDC
Machine Payment      → Quote + Budget + Payment Intent
Settlement/Escrow    → 链上 escrow + delivery proof + dispute
```

MVP：
- Escrow 合约（deposit → deliver → accept/reject → release/refund）
- Quote 生成 + 签名
- Agent A 搜索翻译 → 锁钱 → 自动验收（非空 + 语言检测）
- Agent B 注册服务 → 翻译 → 提交交付
