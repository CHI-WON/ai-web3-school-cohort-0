# 2026-05-18 学习笔记

> 来源：Handbook → AI 基础 → [大语言模型（LLM）](https://aiweb3.school/zh/handbook/ai-basics/llm/)
> 实验代码：`experiments/token_embedding_demo.py`、`experiments/transformer_hallucination_demo.py`

## 学习内容

### 1. LLM 第一性原理

- **核心认知**：LLM 生成的是概率上合理的输出，不是天然可信的事实。模型是推理入口，不是最终验证。
- **三条原则**：
  1. 模型当推理层，不当真相源 —— 关键事实来自数据库、API、日志
  2. 输出变成可检查对象 —— schema、参数、引用，不留在自然语言
  3. 不确定性前移暴露 —— 显式降级，不编顺滑答案

### 2. Token

- Token 是 tokenizer 切分后的片段，≠ 一个字/一个单词
- **实操发现**：EVM 地址 42 字符 → 27 token；驼峰命名 `swapExactTokensForTokens` → 切成 4 个独立 token
- 工程判断：JSON / 代码 / 长标识符比纯文本更吃 token，不能以"字数少"判断是否装得进 context window

### 3. Embedding

- 文本→向量，衡量语义接近度。用于搜索、聚类、RAG
- **边界**：向量相似度 0.94 ≠ 结论正确。两篇相似度 0.94 和 0.92 的文档可以结论完全相反
- 解法：Embedding 是召回起点，后面必须接来源校验 + 时间戳 + 审计状态 + 规则检查

### 4. Transformer

- Attention 机制：模型生成时关注上下文不同位置，但这是**概率分配**，不是精确检索
- **实操演示**：模型可能高度关注数量和价格（"1000 USDC"、"1 ETH = 3500"），但对约束条件（"余额只有 50 USDC"）关注度极低
- 结论：硬约束（余额不足、禁止交互未审计合约）不能靠 attention/prompt，必须用规则引擎

### 5. Hallucination

- 幻觉 = 模型在概率空间生成最合理的错误答案
- **三类幻觉**：
  - 编造 API/参数 → schema 校验 + 函数签名白名单
  - 事实错误 → 来源引用 + 时间戳
  - 过时信息 → 版本检查
- Builder 视角：幻觉不是答案质量问题，是执行风险（错误参数 → 交易失败/资金损失）

### 6. Multimodal

- 让模型读懂图表、审计报告、MetaMask 弹窗、错误堆栈
- **边界**：多模态扩展感知，不扩展验证。截图可伪造/遮挡/PS
- **黄金法则**：涉及资产/权限/签名的判断 —— 必须回到结构化数据（tx hash + 链上查询），不接受截图作为唯一证据

### 7. LLM 在 AI × Web3 中的位置（三层防线）

```
编排层:  LLM（理解意图、生成计划）
数据层:  链上查询（RPC、索引器、合约 ABI）
执行层:  工具调用（钱包签名、合约交互）
安全层:  规则引擎 + schema 校验 + 模拟交易 + 人工确认
```

- Layer 1: LLM 推理入口
- Layer 2: 外部校验（确定性）
- Layer 3: 人工确认（最终裁决）

## 今日纠正

| 错误 | 纠正 |
|------|------|
| Prompt engineering 放在数据层 | 应在**编排层** |
| 链上 RPC 未归类 | 应在**数据层** |
| MetaMask 截图可用于签名判断 | 截图不可信，必须解析交易原始数据 |
| 规则引擎放在编排层 | 应在**安全层**（Guard，不是编排） |
| 链上查询放在执行层 | 应在**数据层**（RPC/索引器是数据源） |

## 收获

- 建立了 LLM 作为系统组件的正确心智模型：推理入口 ≠ 真相源 ≠ 法官
- 通过实操理解了 token 消耗的真实情况（地址、JSON、驼峰命名的 token 密度）
- 清楚了 Embedding 的边界：语义相似 ≠ 事实正确
- 掌握了 Builder 视角下的幻觉风险分类和应对策略
- 建立了 AI × Web3 系统中的四层架构和三层防线

## 遇到的问题

- Transformer attention 和硬约束的关系 —— 经过讨论理解了 attention 是概率性的，安全约束不能放 prompt 里
- Embedding 检索结果遇到矛盾信息时如何处理 —— 理解了向量相似只是召回起点
- 多模态在 Web3 场景中的信任边界 —— 截图不能作为唯一证据
- 四层架构的命名需要跟上 Handbook 标准术语
