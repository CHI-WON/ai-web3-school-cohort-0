## 晚间学习：Bridge 章节推进（23:00 - 01:00）

### Chain-aware Context（链感知上下文）
让 AI 在回答前能「看见」真实链上数据，不靠语言记忆猜。
- 链上事实必须从工具/索引层读取，不能靠模型记忆
- 链上状态有时间性——带 chain id + block number + 读取时间
- 上下文要带来源（tx hash / explorer link），区分事实 vs 解释
- 好上下文包 = 用户目标 + chain id + 地址余额 + 合约ABI文档 + 最近交易 + 索引同步状态 + citation

### Web3 Tool Use（Web3 工具调用）
把 RPC/合约读/交易生成/钱包签名/DeFi 操作变成 Agent 可调用工具。
- 读写分离：读链和写链必须是不同工具、不同权限
- 不要让 Agent 直接拼接任意 calldata
- 权限分层：查询自动 → 小额 session key → 大额人工确认 → 任意合约默认禁止
- 工具返回必须结构化：chain id + method + result + error + timestamp

### Agent Workflow（智能体工作流）
把执行链路组织成可控流程，不让模型自由发挥。
- Task Graph：目标拆成节点和依赖（读余额→查价格→模拟→确认→执行→追踪）
- State Machine：draft → context_loaded → plan_ready → waiting_confirmation → submitted → confirmed
- Human-in-the-loop 分层：只读自动 | 草稿自动 | 小额 session key | 大额人工 | 超 policy 拒绝
- 发交易失败不能盲重试——先查 receipt，确认是否已广播
- Regression Set：每次改模型前跑固定测试集（错误链/无限approve/余额不足/用户拒绝...）

### Agent Wallet（智能体钱包）
不是"给 AI 一个私钥"，是给 Agent 可验证、可限制、可撤销的行动空间。
- Session Key：临时小钥匙，限制时间/合约/金额/token/是否可转外部/随时撤销
- Policy：系统可检查规则——每天最多 X、只调白名单、滑点>1%停、NFT 转出人工
- Guard：确定性拦截层——simulation 成功 ≠ 应该发（如无限 approve）
- Revocation：权限收回——不只是用户关，系统也应在额度用完/异常时自动收紧
- 理想链路：用户约束 → Agent 计划 → 受限交易 → Guard+Simulation → 人工确认 → Safe 执行 → 日志

### Bridge 四层骨架
```
Chain-aware Context  →  输入层：Agent 看见什么
Web3 Tool Use        →  能力层：Agent 能调什么，怎么约束
Agent Workflow       →  流程层：按什么走，不跑飞
Agent Wallet         →  权限层：花多少钱、对谁花、怎么收回
```

这四层直接映射 Hackathon B 方向（Agent Payment/Commerce）的技术基础。
